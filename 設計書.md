# バナー・画像 自動生成ツール 設計書

## 1. 概要

Gemini の画像生成 API を使った Web ツール。ブラウザ上のタブで3つのモードを切り替えて使う。すべてのモードで4大ECモール（Amazon / 楽天市場 / Yahoo!ショッピング / Qoo10）の規約・トーンに合わせた最適化が可能。

1. **セールバナー モード**: 商品情報を入力し、選択モールの推奨バナーサイズ2種を「同一デザイン」で自動生成（モール未選択時は従来の楽天1080×270＋LINE1040×585）。
2. **自由生成 モード**: テキスト指示と複数の参考画像を1項目ずつ設定し、いくつでも追加してまとめて生成。モール選択でトーン・規約を反映。
3. **商品ページ モード**: モール＋商品の特徴＋商品画像を入れると、そのモールの推奨構成（規約・ストーリー）に沿って商品ページ画像を複数枚（モール推奨枚数）自動生成。

- 動作形態: ローカルで起動する Web アプリ（Node.js + Express）
- 画像生成モデル: `gemini-3.1-flash-image`（環境変数で変更可能）
- API バージョン: `v1beta`（`responseModalities` と `imageConfig.aspectRatio` を使用）
- モール定義は `malls.js` に集約（サイズ・1枚目規約・トーン・バナーサイズ・商品ページ構成スロット）

---

## 2. 技術スタック

| 領域 | 技術 | 用途 |
|------|------|------|
| 実行環境 | Node.js (>=18) | サーバー実行。標準の `fetch` を利用するため v18 以上が必須 |
| Web サーバー | Express 4 | 静的ファイル配信 + API エンドポイント |
| 画像生成 | Gemini API (`gemini-3.1-flash-image`) | テキスト/画像からバナー画像を生成 |
| 画像リサイズ | sharp | 生成画像を推奨バナーサイズへリサイズ |
| Zip 生成 | archiver | 複数バナーを 1 つの zip にまとめる |
| pptx 生成 | pptxgenjs | 文字なし背景＋編集可能テキストの PowerPoint を生成 |
| 設定管理 | dotenv | API キー・モデル・ポートを `.env` から読み込み |
| フロントエンド | 素の HTML / CSS / JavaScript | 追加ライブラリ不要のシンプルなフォーム UI |

## 4. セットアップと起動

1. このフォルダで依存をインストール
   ```bash
   npm install
   ```
2. `.env.example` を `.env` にコピーし、Gemini API キーを設定
   ```bash
   cp .env.example .env
   # .env の GEMINI_API_KEY を編集
   ```
   API キーは Google AI Studio（https://aistudio.google.com/apikey）で取得。
3. 起動
   ```bash
   npm start
   ```
4. ブラウザで `http://localhost:3000` を開く。

## 5. UI（2モード・タブ切替）

`public/index.html` は上部のタブで2モードを切り替える。

### 5.1 セールバナー モード

1 商品 = 1 カードとして動的に追加できる。各項目はすべて任意入力。

| 項目 | 入力形式 | 説明 |
|------|----------|------|
| 商品名 | テキスト | バナーに載せる商品名 |
| セール種別 | セレクト | 「ポイントアップ」／「割引」／指定なし |
| 倍率 または 割引率 | テキスト | 例: `10`（倍）/ `30`（%OFF） |
| セール期間 | テキスト | 例: `6/10 0:00 〜 6/15 23:59` |

- 「＋ 商品を追加」で複数商品を入力可能。
- 「×」ボタンで各カードを削除。
- 「バナーを生成」で全商品をまとめて `POST /api/generate` へ送信。

### 5.2 自由生成 モード

セールバナー モードと同様に、**1項目＝1カード**として動的に追加できる（「＋ 項目を追加」）。各カードに以下を設定する。

| 項目 | 入力形式 | 説明 |
|------|----------|------|
| 指示テキスト | テキストエリア | どんな画像を作るかの自由記述（任意） |
| 参考画像 | ファイル（複数可） | クリックまたはドラッグ＆ドロップで追加。サムネイル表示・個別削除可。元画像・参考デザインなどを複数渡せる（任意） |
| アスペクト比 | セレクト | 1:1 / 4:5 / 16:9 / 9:16 / 4:1 / 1:4 / 8:1 / 21:9 など。カードごとに個別指定 |

- 各カードはテキストか参考画像のどちらか一方が必須。各カードにつき画像を1枚生成する。
- 参考画像はブラウザ側で `FileReader` により base64 化。全カードをまとめて `POST /api/freeform-batch` へ送信。
- 結果は項目数ぶんをグリッド表示。各画像を個別ダウンロードできるほか、2件以上のときは「一括 Zip ダウンロード」（`POST /api/zip-free`）でまとめて保存可能。
- サーバー側は最大3並列で生成し（`runPool`）、一部の項目が失敗しても残りは返す。

---

### 5.3 商品ページ モード

モール（必須）・商品の特徴テキスト・商品画像（任意・複数可）を入力し、「商品ページ画像を生成」を押すと、選択モールの推奨構成スロット数ぶんの画像を自動生成する（`POST /api/product-page`）。

- 生成枚数はモール推奨に応じて自動（`malls.js` の `pageSlots.length`）。Amazon=8枚、楽天=8枚、Yahoo=6枚、Qoo10=6枚。
- 各スロットは役割（メイン／ブランド／スペック／比較／レビュー等）が決まっており、その役割に沿ったプロンプトで生成。1枚目（メイン）はモール規約（白背景・文字制限等）を厳守する指示を付与。
- 商品画像は全スロット共通の参照画像として渡す。
- 最大3並列で生成。結果はグリッド表示＋個別／一括Zip（`POST /api/zip-free`、役割名でファイル名付け）。

## 6.5 モールプロファイル（malls.js）

4大モールの違いを `MALLS` オブジェクトに集約。各モールに以下を保持する。

| フィールド | 内容 |
|------------|------|
| `thumbSize` | 推奨サイズ（例: Amazon 1600×1600px以上） |
| `tone` | 訴求トーン（Amazon=ロジカル、楽天=賑わい、Yahoo=スマート、Qoo10=SNS映え） |
| `mainRule` | 1枚目（メイン）の規約（白背景・テキスト20%以下・枠線NG・左上余白など） |
| `bannerSizes` | バナー出力サイズ2種（主・副） |
| `pageSlots` | 商品ページの推奨構成（役割・アスペクト比・指示の配列） |

ヘルパー: `mallGuidance(key)`（バナー・自由生成のプロンプトに規約・トーンを注入）、`buildProductPagePrompt(key, slot, features, i, total, taste)`（商品ページ1スロット分のプロンプト生成）。

## 6.6 デザインテイスト（malls.js / TASTES）

全モード共通で選べる「デザインの勝ちパターン」5分類。`TASTES` にフォント・配色・レイアウトの指示を保持し、`tasteGuidance(key)` でプロンプトに注入する（未選択時はモール標準）。

| キー | テイスト | 概要 |
|------|----------|------|
| `logical` | スタイリッシュ＆ロジカル | 男性向け・機能美。太ゴシック、黒/ネイビー、引き出し線でスペック訴求 |
| `natural` | ナチュラル＆ニュアンス | 女性向け・情緒。明朝/手書き、ベージュ/くすみ、余白と透明感 |
| `luxury` | ラグジュアリー＆ミニマル | 高級・ギフト。細明朝/セリフ、ゴールド/漆黒、文字少なめ・質感重視 |
| `sns` | SNS・トレンド | スマホ・若年層。丸ゴシック/Y2K、ネオン/ホログラム、コラージュ・短文 |
| `impact` | インパクト＆大文字 | 視認性・メガモール。極太/角ポップ、赤×黄、座布団帯・実績バッジ |

各モード（バナー・自由生成・商品ページ）のリクエストに任意の `taste` を付与でき、モール最適化と掛け合わせて反映される。

## 6.6.1 商品ジャンル（malls.js / GENRES）

デザインテイストとは別軸で、商品ジャンルごとの「勝ちパターン」を10種類定義（`GENRES`）。各ジャンルに写真の見せ方・フォント・配色・入れるべき要素をまとめた `guide` を持ち、`genreGuidance(key)` でプロンプトへ注入する。

食品・グルメ（シズル＆産直）／コスメ・美容（肌悩み解決＆クリーン）／健康食品・サプリ（エビデンス＆情報密度）／アパレル・ファッション（雑誌・LOOKBOOK）／インテリア・寝具（空間調和）／家電・ガジェット（近未来＆スペック）／日用品・雑貨（お悩み解決＆時短）／ベビー・キッズ・ペット（安心安全）／スポーツ・アウトドア（アクティブ）／ビジネス・文具・ギフト（信頼・フォーマル）。

モール・テイスト・ターゲット・ジャンル・備考は `extrasGuidance({genre,taste,target,notes})` で1つのガイド文字列にまとめ、各プロンプトへ注入する（すべて任意・掛け合わせ可）。

## 6.7 共通の追加指定（備考・ターゲット）と画像個別機能

全モード共通で、モール・テイストに加えて以下を指定・操作できる。

- **備考・詳細指示（notes）**: 自由記述（例「添付のモデルを使って」「夜のテイストで」）。プロンプトに `【追加指示・備考】` として注入。
- **ターゲット年齢層（target）**: 10代/20代/30〜40代/40代/50代以上の5段階（`malls.js` の `TARGETS`）。配色・フォント・コピー・モデル選定に反映。
- テイスト・ターゲット・備考は `extrasGuidance({genre,taste,target,notes})` で1つのガイド文字列にまとめ、各プロンプトへ注入する。
- **スタイル見本画像（styleImages）**: 全タブに「スタイル見本画像（複数可）」の枠を用意。商品画像とは別枠で、配色・フォント・装飾・雰囲気を見本に寄せる。見本内の文字・数値・情報はコピーしない（文言・商品情報は特徴テキストと商品画像のみから取得）。
- **スタイル仕様（styleSpec）の確認・編集**: 参照画像をそのまま渡すだけではこの画像生成モデルがテイストを汲み取りにくいため、`describeStyleImages()`（`GEMINI_TEXT_MODEL`）で見本を解析し、配色・フォント・装飾・レイアウト等を文章の「デザイン仕様」に変換してプロンプトへ最優先で注入する。`POST /api/analyze-style` で画面に仕様を表示・編集でき、生成時に `styleSpec` が指定されていればそれを優先（空ならその場で自動解析）。`styleParts(styleImages, styleSpec)` が指示文＋画像パーツを生成する。

生成後の各画像カードには次の操作が付く。

- **③ 1枚ごとの修正**: 修正指示を入力して「修正」を押すと、その画像を参照に Gemini が再編集（`POST /api/refine`）。結果は同じカードに差し替わり、Zip にも反映される。
- **④ 編集可能データ（pptx）書き出し**: 「編集pptx」を押すと、(1) 元画像から文字を除去した『背景版』を生成、(2) 元画像内の文字を `GEMINI_TEXT_MODEL`（既定 `gemini-2.5-flash`）で抽出、の2つを並行実行し、背景版の上に「抽出した元の文字」を編集可能テキストボックスとして載せた PowerPoint を出力する（`POST /api/export-pptx`）。元の文言が入った状態で PowerPoint で自由に編集できる。AI 画像は文字が焼き込まれるため、背景と文字を分離してこの形で提供している。文字抽出に失敗した場合はプレースホルダーにフォールバックする。

## 6. 処理フロー

### 6.1 全体の流れ

```
[ブラウザ]                       [Express サーバー]                 [Gemini API]
  入力フォーム
    │  POST /api/generate
    │  { products: [...] }  ───────►
    │                          各商品ごとに:
    │                          ① 楽天 4:1 を生成     ──────────────►  画像(4:1)生成
    │                          ② ①を参照に LINE 16:9  ──────────────►  画像(16:9)生成
    │                          ③ sharp でリサイズ
    │                             楽天→1080×270 / LINE→1040×585
    │  ◄───────  results[] (base64 PNG×2)
  プレビュー表示・個別DL
    │  POST /api/zip { items:[...] }  ─►  archiver で zip 化
    │  ◄───────  banners.zip
  一括ダウンロード
```

### 6.2 ステップ詳細

**① Gemini API で画像生成（`server.js` の `callGemini`）**

- エンドポイント:
  `POST https://generativelanguage.googleapis.com/v1/models/{MODEL}:generateContent`
- 認証: HTTP ヘッダー `x-goog-api-key`（`.env` の `GEMINI_API_KEY`）
- リクエスト本文の要点:
  ```json
  {
    "contents": [{ "parts": [{ "text": "<デザインプロンプト>" }] }],
    "generationConfig": {
      "responseModalities": ["TEXT", "IMAGE"],
      "responseFormat": { "image": { "aspectRatio": "4:1" } }
    }
  }
  ```
- リクエスト本文の要点（`server.js` の `generateImageFromParts`）:
  ```json
  {
    "contents": [{ "parts": [ {"text": "..."}, {"inline_data": {"mime_type": "image/png", "data": "<base64>"}} ] }],
    "generationConfig": {
      "responseModalities": ["TEXT", "IMAGE"],
      "imageConfig": { "aspectRatio": "4:1" }
    }
  }
  ```
- `parts` に `text` と任意個の `inline_data`（参考画像）を入れられる。バナー・自由生成の両モードで共通の低レベル関数 `generateImageFromParts(parts, aspect)` を使う。
- レスポンスの `candidates[0].content.parts[].inlineData.data`（base64 PNG）を取り出す。
- ※ API バージョンは `v1beta`。`v1` では `responseModalities` / `imageConfig` が受け付けられないため注意。

**② 楽天 4:1 と LINE 16:9 を「同じデザイン」で生成（`generateBanners`）**

- まず楽天用に **アスペクト比 4:1** で 1 枚生成する。
- 次に LINE 用は、**生成した楽天画像を参照画像（`inline_data`）として渡し**、「同じデザインのまま 16:9 に再構成せよ」というプロンプトで **アスペクト比 16:9** を生成する。これにより配色・スタイル・文言の統一感を保つ。
  - これは `gemini-3.1-flash-image` の画像編集（テキスト+画像→画像）機能を利用。
  - アスペクト比は `responseFormat.image.aspectRatio` で指定（モデルは 4:1・16:9 を含む複数比に対応）。

**③ 推奨バナーサイズへリサイズ（sharp）**

- 楽天: `1080 × 270 px`
- LINE: `1040 × 585 px`
- `sharp(buf).resize(w, h, { fit: "fill" })` で指定サイズちょうどに合わせる。`fit: "fill"` は縦横比を無視して枠ぴったりに伸縮するが、生成時に同じアスペクト比（4:1 / 16:9）で作っているため歪みは最小限。

**④ クライアントへ返却・プレビュー**

- リサイズ後の PNG を base64 で JSON に格納して返す。
- フロントは `data:image/png;base64,...` として `<img>` に表示し、`<a download>` で個別保存可能。

**⑤ 一括 Zip ダウンロード（`/api/zip` + archiver）**

- フロントが保持している生成済み base64 画像を `POST /api/zip` に送信。
- サーバーは archiver でストリーム zip を作り、
  `{商品名}_楽天_1080x270.png` / `{商品名}_LINE_1040x585.png` という名前で格納して返す。

## 7. API 仕様

### `POST /api/generate`（バナー）
- リクエスト: `{ "products": [{ "name", "saleType", "rate", "period" }, ...], "mall": "rakuten" }`（`mall` は任意。未指定なら楽天＋LINE）
- レスポンス:
  ```json
  {
    "results": [
      {
        "index": 0, "name": "安全なファイル名", "displayName": "表示名", "ok": true,
        "banners": [
          { "key": "main", "label": "楽天バナー", "size": "1080x270", "image": "<base64>" },
          { "key": "sub",  "label": "LINE/縦長バナー", "size": "1040x585", "image": "<base64>" }
        ]
      }
    ]
  }
  ```
- `banners` は選択モールの `bannerSizes` に応じて可変。商品単位でエラーを握りつぶし、一部失敗しても他商品は生成される。

### `GET /api/malls`
- 利用可能なモール一覧 `{ malls: [{ key, label, thumbSize, pageCount }] }` を返す。

### `POST /api/product-page`（商品ページ）
- リクエスト: `{ "mall": "amazon", "features": "商品の特徴…", "images": [{ "data": "<base64>", "mimeType": "image/png" }] }`
- レスポンス: `{ "ok": true, "mall": "Amazon", "count": 8, "results": [ { "ok": true, "image": "<base64>", "role": "メイン画像…", "aspect": "1:1" }, ... ] }`
- モール必須。特徴テキストか商品画像のいずれか必須。スロット単位で失敗は `ok:false`。

### `POST /api/freeform-batch`（自由生成・複数項目）
- リクエスト:
  ```json
  {
    "mall": "rakuten",
    "items": [
      { "prompt": "指示テキスト", "aspect": "4:1", "images": [ { "data": "<base64>", "mimeType": "image/png" } ] },
      { "prompt": "別の指示", "aspect": "16:9", "images": [] }
    ]
  }
  ```
- `mall` は任意。指定すると各項目のプロンプトにモールの規約・トーンを注入する。
- レスポンス: `{ "ok": true, "results": [ {"ok": true, "image": "<base64>", "aspect": "4:1"}, {"ok": false, "error": "..."} ] }`
- 各項目で画像を1枚生成。最大3並列で実行し、失敗した項目は `ok:false` で返す（全体は止めない）。

### `POST /api/freeform`（単発・後方互換）
- リクエスト: `{ "prompt", "aspect", "count"(1-10), "images":[...] }`
- レスポンス: `{ "ok": true, "aspect", "count", "results":[{ok,image}...] }`
- 同じ指示から `count` 枚をまとめて生成する単発版。現在のUIは `/api/freeform-batch` を使用。

### `POST /api/refine`（1枚修正）
- リクエスト: `{ "image": "<base64>", "instruction": "修正指示", "aspect": "4:1" }`
- レスポンス: `{ "ok": true, "image": "<base64>" }`（修正後の画像）

### `POST /api/export-pptx`（編集可能データ書き出し）
- リクエスト: `{ "title": "name", "items": [ { "image": "<base64>", "aspect": "4:1", "texts": ["商品名", ...] } ] }`
- レスポンス: `name.pptx`（各 item＝1スライド。文字なし背景画像＋編集可能テキストボックス）

### `POST /api/zip-free`
- リクエスト: `{ "items": [ { "data": "<base64>", "name": "任意" }, ... ] }`
- レスポンス: `generated_images.zip`（`01_xxx.png` …）

> 補足: `/api/generate`・`/api/freeform-batch`・`/api/product-page` はいずれも任意で `taste`・`target`・`notes` を受け取り、プロンプトに反映する。

### `POST /api/zip`
- リクエスト: `{ "items": [{ "name", "rakuten", "line" }, ...] }`
- レスポンス: `banners.zip`（`application/zip`）

### `GET /api/health`
- 稼働確認用。`{ ok, model, hasKey }` を返す。

## 8. プロンプト設計（`buildDesignPrompt`）

- 入力された項目（商品名・セール種別・倍率/割引率・セール期間）を日本語の箇条情報に整形。
- 「プロのECバナーデザイナー」という役割指定に加え、以下のデザイン要件を明示してテキスト崩れや訴求力不足を防ぐ:
  - 日本語テキストを正確・大きく配置
  - セール種別・倍率/割引率を最も目立たせる
  - 鮮やかでモダンな配色・レイアウト
  - 綴り間違いの防止
- すべて空欄の場合は「おまかせのセールバナー」として生成する。

---

## 9. セキュリティ・運用上の注意

- API キーはサーバー側の環境変数（`.env`）でのみ保持し、ブラウザには露出しない。
- `.env` は公開リポジトリにコミットしない（`.env.example` のみ共有）。
- Gemini API の利用は従量課金。商品数 × 2 枚（楽天 + LINE 再構成）分の生成が走る点に留意。
- 生成画像には Google の利用ポリシーが適用される（他者の権利を侵害する内容の生成は禁止）。

## 10. カスタマイズポイント

| 変更したいこと | 変更箇所 |
|----------------|----------|
| 使用モデル | `.env` の `GEMINI_MODEL` |
| バナーの推奨サイズ | `server.js` の `SIZES` |
| 生成プロンプトの文言 | `server.js` の `buildDesignPrompt` / `generateBanners` |
| リサイズ方法（歪み回避） | `server.js` の `sharp(...).resize(...)` の `fit` オプション（例: `cover` + `position`） |
| 入力フォーム項目 | `public/index.html` の `productTemplate` |
